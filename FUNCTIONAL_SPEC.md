# Franmarine — Digital Worksheet Package

## Functional Specification

A single-page, offline-capable web application that captures a complete day of
commercial diving / marine operations on one device. The package consolidates
the paper forms a supervisor would normally complete (project sheet, dive log,
toolbox meeting, SWMS sign-off, emergency drill report, permits) into one
synchronised document and exports a print-ready PDF and a CSV of the dive log.

The application is implemented as static HTML, CSS and JavaScript with no
backend. State persists to `localStorage` and can be exported / re-imported as
a JSON file.

---

## 1. Application Architecture

| Layer | File | Purpose |
|-------|------|---------|
| Markup | `index.html` | App shell, navigation, all six form pages, floating action panels |
| Styling | `styles/01.css` | Layout, theming, print stylesheet |
| Logic | `scripts/03.js` | Page initialisation, dynamic rows, calculations, persistence, export |

### 1.1 Shell Components

- **Header** — Brand mark, hamburger button (`toggleSidebar`), Options
  dropdown (`toggleOptions`).
- **Sidebar / Pill Tabs** — Two synchronised navigation surfaces. Each entry
  carries a `data-tab` attribute that maps to a `#page-<tab>` form page.
- **Form Pages** — Six top-level views, only one shown at a time
  (`.form-page.active`).
- **Floating Quick-Log Panel** — Always-available activity logger
  (`#floatLog`).
- **Save Indicator** — Toast that fades in for ~1.2 s after each auto-save.

### 1.2 Navigation Functions

| Function | Behaviour |
|----------|-----------|
| `switchTab(tab)` | Activates the matching `.form-page`, sidebar button and pill tab; auto-collapses the sidebar on screens < 768 px. |
| `toggleSidebar()` | Toggles `.closed` on `#sidebar`. |
| `toggleOptions()` | Shows / hides the Options dropdown (PDF, CSV, Save / Load JSON, Clear All, Diving Mode). |
| `toggleDivingMode()` / `applyDivingMode()` | Switches the worksheet between **Diving** and **Non-Diving** modes. In non-diving mode the Diving Log page and its tab buttons are hidden, and if the active page was the dive log it falls back to Daily Activities. |

## 2. Page-by-Page Functional Reference

### 2.1 Daily Activities (`#page-dailyActivities`)

Acts as the master entry point — project metadata entered here syncs to every
other page via `syncProj()` (any input with `data-proj="<key>"`).

#### 2.1.1 Region Selector
- Buttons **NSW** / **WA** call `setRegion(r)`.
- Side effects defined by `REGION_CONFIG`:
  - Sets the hidden `data-proj="region"` value (synced everywhere).
  - Auto-fills **Nearest Hyperbaric Chamber** + **Chamber Phone**:
    - NSW → Prince of Wales Hospital, Randwick — (02) 9382 3880.
    - WA → Fiona Stanley Hospital, Murdoch — (08) 6152 5222.
  - Repopulates the **Hat Model / Serial** dropdowns from `HAT_INVENTORY`
    (Perth or Sydney) via `updateHatModelOptions()` /
    `updateHatSerialOptions(diverNum)`.

#### 2.1.2 Project Description
Synced fields: Project, Date, Job No., Location, Vessel/Facility, Client,
Supervisor, Client Contact, Contact Number. Plus local-only Started at
Workshop / Arrived Onsite times.

#### 2.1.3 Team Roster
- `addTeamMember(name?)`, `removeTeamMember(i)`, `updateTeamMember(i, v)`
  manage the in-memory `teamMembers` array.
- `renderTeamInputs()` redraws the roster.
- `refreshAllSignSections()` rebuilds the sign-on rows on every other page
  (Fit-to-Dive, Toolbox, SWMS, Emergency Drill) so each member always has a
  signature canvas. It also refreshes datalists (`hatCheckList1/2`,
  `diveNameList`, `diverReviewList`).

#### 2.1.4 Pre-Dive Checks
LP / HP air, Oxyviva, AED, Comms tested, CCTV tested, Diver 1/2 HAT-check
performer (datalist of team), Diver 1/2 hat model + serial (filtered by
region), chamber + chamber phone.

#### 2.1.5 Onsite Risk Assessment
A "Sign as Fit to Dive" panel rendered into `#dl_fitToDive` — one signature
canvas per team member.

#### 2.1.6 Daily Activity Log
- Table with **20 starter rows** generated by `makeRows('da_activities', 20, …)`.
- `addActivityRow()` inserts a blank time + description row.
- `sortActivityLog()` re-orders all rows chronologically by the time input
  (blank times sink to the bottom). Triggered after every insert.
- Receives **auto-rows** from other modules:
  - `.auto-dive-row` from `syncDivesToActivity()` — each dive's *left
    surface*, *left bottom*, *arrived surface* events, coloured by diver.
  - `.auto-drill-row` from `syncDrillToActivity()` — `DIVING DRILL
    COMPLETED — <type>` line stamped at the last filled drill-step time.
- Auto-row description text is editable; edits are remembered in
  `autoRowEdits` keyed by `d<diverIdx>s<subIdx>_<event>` so they survive a
  re-sync.

### 2.2 Diving Log (`#page-divingLog`)

#### 2.2.1 Diver Entries
- `init()` creates **8 diver containers** (`dive-entry`, indexed 0–7), each
  with its own colour from `_diverColors`.
- The first two entries are open by default; the rest are collapsed and
  toggle on summary click.
- Each entry contains a **Diver Name** input (datalist `diveNameList` of
  team members) plus a `dive-subs` container with one or more sub-dives.

#### 2.2.2 Sub-Dive Rows
- `createDiveSubEntry(diverIdx, subIdx, container)` adds a sub-dive with
  three rows of fields:
  1. Surface Interval (datalist `siDatalist`), RF (`RF_OPTIONS` 1.0–2.0),
     Bail Out Pressure (Bar).
  2. Left Surface (time), Depth (m), NDL, Left Bottom (time), Arrived
     Surface (time).
  3. Total Dive Time, Total Bottom Time, RG (`RG_OPTIONS` A–O).
- `addDiveToEntry(diverIdx)` ("+ Add Dive") appends another sub-dive.
- `reindexDiveSubs(diverIdx)` re-numbers sub-dive labels after a removal.

#### 2.2.3 Calculations
- `calcDiveTotalTimes()` runs after any field change:
  - **Bottom Time** = Left Bottom − Left Surface (minutes; wraps past midnight).
  - **Total Dive Time** = Arrived Surface − Left Surface (minutes; wraps
    past midnight).
  - Either auto-fill is suppressed once the user manually edits that field
    (`dataset.userEdited`).

#### 2.2.4 Sync to Activity Log
- `syncDivesToActivity()` rebuilds all `.auto-dive-row` rows in the daily
  log from the current dive data, sorted by left-surface time, and refreshes
  each entry's collapsed summary line (`Diver N — Name @ Depth m [time]` or
  `Diver N (3 dives)`).

### 2.3 Toolbox Meeting (`#page-toolbox`)

- **Project Description** — duplicate `data-proj` fields that mirror page 1.
- **Scope of Work Discussed** — free text.
- **Attendees — Sign On** (`#tb_signOn`) — one signature canvas per team
  member.
- **Hazard Identification & Controls** — 8 rows generated by
  `makeHazardRows('tb_hazards', 8)` (Hazard / Control / By Whom).
- **Requirements Checklist** (`#tb_checklist`) — 10 fixed items rendered with
  Yes / No / N/A selects. The select gets a `cl-yes` / `cl-no` / `cl-na`
  class on change for colour coding. The final item *CONSIDERED SAFE TO
  PROCEED* is bolded.
- **Authorisation** — Supervisor and Diver signature canvases (`tb_supSig`,
  `tb_diverSig`) with **Clear** buttons (`clearSig(id)`).

### 2.4 SWMS Sign-Off (`#page-swms`)

- **Upload zone** — click or drag-and-drop a PDF
  (`handleSWMSUpload(input)`):
  - Validates MIME type, defers if `pdfjsLib` isn't ready.
  - Renders every page to an on-screen canvas via PDF.js
    (`renderSwmsPages()`).
  - Auto-fills SWMS Title from the file name when blank.
- **Remove PDF** — `removeSWMS()` clears the doc and the rendered pages.
- **SWMS Details** — Title, Revision, Date, etc.
- **Team Sign-Off** (`#swms_signOff`) — signature canvas per team member,
  preceded by a "I have read and understood this SWMS" disclaimer.

### 2.5 Emergency Drill Report (`#page-emergencyDrill`)

- **Attendees — Sign On** (`#ed_attendees_list`) — signature per team member.
- **Drill Type** — free text; drives the auto-row in the activity log.
- **Emergency Drill Steps** (`#ed_steps`) — 15 rows of Time + Step. Each
  input fires `syncDrillToActivity()` so the activity log stays current.
- **Comments / Action Plan** (`#ed_actions`) — 5 rows of Comment / Who /
  When (date).

### 2.6 Permits (`#page-permits`)

- **Project Description** — synced via `data-proj`.
- **Upload zone** — accepts multiple PDFs and images
  (JPEG/PNG/GIF/TIFF/HEIC). Drag-and-drop wired up in
  `refreshAllSignSections()`.
- `handlePermitsUpload(input)` iterates files sequentially and dispatches:
  - **PDFs** → `renderPermitDoc(pdf, name, cb)` renders all pages with
    PDF.js into a removable wrapper.
  - **Images** → `renderPermitImage(file, cb)` renders a `<img>` preview.
- Each permit wrapper has its own **Remove** button. `removePermits()`
  clears all uploads, and `updatePermitsStatus()` keeps the upload-zone
  label and the master Remove button in sync.

## 3. Cross-Cutting Features

### 3.1 Floating Quick-Log

The persistent **📋 Log Activity** button (`toggleFloatLog()`) opens a panel
that streams entries directly into the Daily Activity Log (`addFloatLog()`),
without leaving the current page.

- Defaults to the current `HH:MM` if no time is entered.
- **Diver hotkey** — leading `D1`–`D8` is stripped from the description and
  rendered as a coloured pill matching the corresponding `dive-entry`
  (`_diverColors`). The pill text is `Diver N — <name>` when a name is set.
- After insert, the new row is sorted into the activity log by time, the
  description input is cleared, and a "✓ Added at HH:MM" confirmation
  flashes for 2 s.
- A one-time **Quick Tip bubble** (`showHotkeyBubble()` /
  `dismissHotkeyBubble()`) explains the D1–D8 hotkey; dismissal is stored
  under `franmarine_hotkey_seen` in `localStorage`.

### 3.2 Signature Capture

`initSigCanvas(id)` wires each `<canvas>` for both mouse and touch input.
Strokes are drawn with a 2 px round cap and persist across saves.

- `clearSig(id)` blanks a single signature.
- `isSigEmpty(canvas)` inspects the alpha channel to decide whether a
  canvas should be considered blank.
- `createSigRow(name, prefix)` produces a labelled row containing the
  signer's name, the canvas and a Clear button. Empty roster slots are
  rendered as `(empty slot)` and hidden from print output.

### 3.3 Auto-Save / Restore (`localStorage`)

- Storage key: `franmarine_ws_<sanitised-pathname>`.
- `scheduleSave()` debounces saves to ~1 s after the last input, change,
  or signature stroke (`pointerup` / `touchend`).
- `saveState()` snapshots:
  - All `<input>` (non-file) / `<textarea>` / `<select>` values, grouped
    per `.form-page`.
  - Team roster, diving-mode flag, `autoRowEdits` overrides.
  - Signatures encoded as PNG data URLs (static + dynamic groups).
- `loadState()` runs ~200 ms after init, restores in order:
  1. Diving mode toggle.
  2. Team roster (which rebuilds sign rows).
  3. Auto-row description edits.
  4. All inputs by page index.
  5. Project field cross-sync.
  6. Signature images.
  7. Region cascade (region → chamber + hat options), preserving any
     user-edited chamber/phone/hat values.
  8. Re-runs `syncDivesToActivity()`, `syncDrillToActivity()` and
     `sortActivityLog()`.

### 3.4 Save / Load JSON File

- `saveToFile()` builds the same state object (`buildStateObject()`) and
  downloads it as `<JobNo>_Worksheet_<Date>.json` (falls back to
  `Franmarine_Worksheet.json` when those fields are blank).
- `loadFromFile(input)` reads a JSON file, confirms the destructive replace,
  applies it via `applyState()`, and mirrors it to `localStorage` so
  auto-save continues seamlessly.

### 3.5 Clear All

`clearAll()` (after a confirm prompt):
- Removes the localStorage entry.
- Clears every input / textarea / select and strips checklist colour
  classes.
- Clears every canvas and resets the team roster.
- Removes uploaded SWMS and permits.
- Drops all auto-generated dive / drill rows and `autoRowEdits`.

### 3.6 Export — PDF (Print)

`exportPDF()` performs a print-to-PDF flow:
1. Sets `document.title` to `<JobNo> - Daily Worksheet - <Date>` so the
   browser's print dialog suggests a meaningful filename.
2. Activates **all** form pages so they render in the printout.
3. Calls `hideEmptyForPrint()` to suppress noise:
   - Daily Activity Log: rows with no values (excluding auto rows).
   - Dive entries with no data.
   - Empty hazard / drill-step / drill-action rows.
   - Empty signature rows where the signer slot is unnamed.
   - Checklist items with no Yes/No/N/A selection.
4. Triggers `window.print()`.
5. After printing, `restoreAfterPrint()` removes the `print-hide` flags and
   the originally-active tab is reactivated.

### 3.7 Export — Dive Log CSV

`exportDiveCSV()` writes a CSV containing project metadata followed by one
row per sub-dive:

```
Diver # | Diver Name | Dive # | Surface Interval | RF | Bail Out Pressure
       | Left Surface | Depth | NDL | Left Bottom | Arrived Surface
       | Total Dive Time | Total Bottom Time | RG
```

Empty divers (no name and no dive data) are skipped. The download filename
follows `<JobNo>_Dive_Log_<Date>.csv` (with sensible fallbacks). Values are
quote-escaped through `csvEsc()`.

### 3.8 Diving / Non-Diving Mode

`toggleDivingMode()` flips a flag persisted with the rest of state. In
non-diving mode the Diving Log page and its sidebar / pill buttons are
hidden, and if it was the active page the user is moved to Daily Activities.

## 5. External Dependencies

- **PDF.js** (`window.pdfjsLib`) — used to render the SWMS document and
  any uploaded permit PDFs to canvas. Both upload handlers defer until
  `window._pdfjsReady` is true.
- No other runtime dependencies; the rest of the app is plain ES5 / DOM
  APIs.

## 6. Persistence Summary

| Mechanism | Trigger | Scope |
|-----------|---------|-------|
| `localStorage[franmarine_ws_…]` | Debounced auto-save on input / change / signature stroke | Full worksheet state including signatures (PNG data URLs) |
| `localStorage[franmarine_hotkey_seen]` | Dismissing the quick-log tip bubble | UI preference only |
| JSON file download | `saveToFile()` from the Options menu | Full state, portable between devices / browsers |
| JSON file upload | `loadFromFile()` from the Options menu | Replaces current state and primes localStorage |

---

## 7. Responsive / Print Behaviour

- Sidebar auto-collapses below 768 px and after each `switchTab` on mobile.
- Pill tabs supplement the sidebar on small screens.
- Print stylesheet (in `styles/01.css`) plus `hideEmptyForPrint()` ensure
  exported PDFs only contain populated rows, used signatures, and
  filled-in checklist items, while still appending the rendered SWMS and
  permit pages at full resolution.
